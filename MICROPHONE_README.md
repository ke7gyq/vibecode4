# PDM Microphone Integration - Implementation Guide

## Overview

This document describes the PDM-to-PCM microphone integration for the Vibecode4 project. The system captures audio from an ATSAMD21  MEMS microphone connected via PDM (Pulse Density Modulation) protocol and converts it to PCM (Pulse Code Modulation) for processing.

## Hardware Setup

### Microphone Connections
- **GPIO 6 (GP6)**: PDM Clock Output (generated by RP2350)
- **GPIO 7 (GP7)**: PDM Data Input (from ATSAMD21 microphone)

### PDM Protocol
- **Clock Rate**: ~4 MHz (configurable in microphone.c via `sm_config_set_clkdiv`)
- **Data Format**: 1-bit PDM stream at high clock rate
- **Conversion Ratio**: 64:1 decimation to produce PCM audio samples

## Architecture

### Component Overview

```
┌─────────────────────────────────────────────────────────────┐
│                   Microphone System                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Hardware (PIO + DMA)                                       │
│  ├─ PIO State Machine 0: Clock generation + data sampling  │
│  └─ DMA Channel 0: Transfer PDM data to memory             │
│                                                             │
│  Microphone Task (Producer)                                │
│  ├─ Reads PDM data from DMA buffer                         │
│  ├─ Converts PDM to PCM (decimation filter)                │
│  ├─ Fills ping-pong buffers (buffer1, buffer2)            │
│  └─ Signals consumer via binary semaphore                 │
│                                                             │
│  Consumer Task(s) (Waiting on semaphore)                  │
│  ├─ Blocked until buffer ready                            │
│  ├─ Reads g_audioReady to identify buffer                 │
│  ├─ Processes audio (DSP, ML, streaming, etc.)            │
│  └─ Calls clear_audio_ready() when done                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Global Variables and API

### Global State
```c
extern volatile uint8_t g_audioReady;        /* 0=none, 1=buffer1, 2=buffer2 */
extern AudioBuffers_t g_audioBuffers;        /* Ping-pong buffer pair */
extern SemaphoreHandle_t g_audioReadySemaphore;  /* Signaling semaphore */
```

### Buffer Structure
```c
typedef struct {
    int16_t buffer1[1024];  /* 1024 int16_t samples */
    int16_t buffer2[1024];  /* 1024 int16_t samples */
} AudioBuffers_t;
```

Each buffer holds 1024 PCM samples (int16_t). With 16-bit samples at common audio rates:
- At 16 kHz: ~64 ms of audio per buffer
- At 31.25 kHz: ~32.8 ms of audio per buffer

## Task Structure

### Microphone Task
**Priority:** 2 (configurable)  
**Stack Size:** 2048 bytes  
**Behavior:** Producer - captures and converts PDM to PCM

```c
void microphone_task(void *parameters)
{
    // 1. Initialize PIO and DMA
    // 2. Start PDM clock generation and data sampling
    // 3. Loop:
    //    - Wait for DMA buffer from PIO
    //    - Collect 32-bit PDM words (64 bits sampled per word)
    //    - Convert group of PDM bits to single PCM sample
    //    - Fill ping-pong buffer
    //    - When buffer full: post semaphore, update g_audioReady, switch buffer
}
```

### Audio Consumer Task
**Priority:** 2 (configurable)  
**Stack Size:** 2048 bytes  
**Behavior:** Consumer - waits for buffers and processes audio

```c
void audio_consumer_task(void *parameters)
{
    while (1) {
        // 1. Block on g_audioReadySemaphore (5 second timeout)
        xSemaphoreTake(g_audioReadySemaphore, pdMS_TO_TICKS(5000));
        
        // 2. Check which buffer is ready
        uint8_t buffer_id = get_audio_ready();  // Returns 1 or 2
        
        // 3. Get pointer to the ready buffer
        int16_t *p_buffer = (buffer_id == 1) ? g_audioBuffers.buffer1 
                                              : g_audioBuffers.buffer2;
        
        // 4. Process audio (DSP, ML, streaming, etc.)
        // ... actual processing ...
        
        // 5. Signal that buffer has been consumed
        clear_audio_ready();
    }
}
```

## Consumer Implementation Example

### Basic Pattern
```c
void my_audio_processor_task(void *param) {
    while (1) {
        // Wait for audio buffer
        if (xSemaphoreTake(g_audioReadySemaphore, pdMS_TO_TICKS(1000)) == pdTRUE) {
            // Get ready buffer
            uint8_t buf_id = get_audio_ready();
            int16_t *p_audio = (buf_id == 1) ? g_audioBuffers.buffer1 
                                              : g_audioBuffers.buffer2;
            
            // Process the samples
            for (int i = 0; i < 1024; i++) {
                int16_t sample = p_audio[i];
                // ... process sample ...
            }
            
            // Signal done
            clear_audio_ready();
        }
    }
}
```

### Advanced: Multiple Consumers
If multiple tasks need to access the same buffer, add a mutex:

```c
static SemaphoreHandle_t g_audioBufferMutex = NULL;

void setup(void) {
    g_audioBufferMutex = xSemaphoreCreateMutex();
    // Create consumer tasks...
}

void my_consumer(void *param) {
    while (1) {
        if (xSemaphoreTake(g_audioReadySemaphore, pdMS_TO_TICKS(1000)) == pdTRUE) {
            // Lock buffer access
            if (xSemaphoreTake(g_audioBufferMutex, pdMS_TO_TICKS(100)) == pdTRUE) {
                uint8_t buf_id = get_audio_ready();
                int16_t *p_audio = (buf_id == 1) ? g_audioBuffers.buffer1 
                                                  : g_audioBuffers.buffer2;
                
                // Make a copy for processing (optional)
                int16_t local_copy[1024];
                memcpy(local_copy, p_audio, sizeof(local_copy));
                
                xSemaphoreGive(g_audioBufferMutex);
                
                // Now process local_copy at own pace
                // ...
            }
            clear_audio_ready();
        }
    }
}
```

## PDM to PCM Conversion

### Algorithm: Decimation-by-64 with CIC Filter

The PDM-to-PCM conversion uses a simplified Cascaded Integrator-Comb (CIC) filter:

1. **Accumulation**: Count set bits (1s) in groups of 64 PDM bits
2. **Decimation**: Decimate to produce one PCM sample per 64 PDM bits  
3. **Scaling**: Convert bit count to signed 16-bit PCM range

### Bit Counting (CIC Integral Stage)
```
For each group of 64 PDM bits:
    ones_count = count_set_bits(pdm_data)
    max_possible = 64
    
    // Map bit count to PCM value
    // High ones_count → positive signal
    // Low ones_count → negative signal
    pcm_value = (ones_count - 32) << 9  // Scale to int16 range
```

### PDM Rate to PCM Rate
- **PDM Clock**: ~4 MHz (configurable)
- **Decimation**: 64:1
- **PCM Sample Rate**: 4 MHz / 64 = 62.5 kHz
- **Actual Rate**: ~31-62.5 kHz depending on clock divider

Configure via `sm_config_set_clkdiv()` in [microphone.c](microphone.c):
```c
// Current: clock_div = 31.25 → ~4 MHz PDM → 62.5 kHz PCM
// For slower: clock_div = 62.5 → ~2 MHz PDM → 31.25 kHz PCM
// For faster: clock_div = 15.625 → ~8 MHz PDM → 125 kHz PCM
```

Create your own consumer task that processes the audio buffers:
```c
void my_processor_task(void *param) {
    while (1) {
        if (xSemaphoreTake(g_audioReadySemaphore, pdMS_TO_TICKS(5000)) == pdTRUE) {
            uint8_t buf_id = get_audio_ready();
            int16_t *p_audio = (buf_id == 1) ? g_audioBuffers.buffer1 
                                              : g_audioBuffers.buffer2;
            
            // Process audio samples
            // e.g., DSP filtering, ML inference, transmission, etc.
            
            clear_audio_ready();
        }
    }
}
```

### Check Buffer Values
The example consumer task prints RMS level of each buffer. For real microphone data, RMS should vary with audio input. Test with:
- Silent environment: RMS near 0
- Loud speech: RMS >10000
- Noise: RMS varies

### Verify Semaphore
If consumer task never wakes up:
- Check `xSemaphoreTake()` return value
- Verify microphone task is running
- Check if `xSemaphoreGive()` is being called

## Performance Considerations

### Memory Usage
```
Buffers: 1024 * 2 * sizeof(int16_t) = 4 KB
DMA temporary buffer: 256 * sizeof(uint32_t) = 1 KB  
Total: ~5 KB
```

### CPU Usage
- **PIO + DMA**: Minimal (hardware handles data transfer)
- **Microphone Task**: ~1-2% CPU for PDM→PCM conversion
- **Consumer Task**: Depends on your processing

### Latency
- From microphone to buffer available: ~16-32 ms (buffer fill time)
- Jitter: <1 ms (PIO/DMA is synchronous)

## Troubleshooting

### Problem: Consumer task always times out
- Check microphone task is running (`printf` in microphone_task loop)
- Verify `xSemaphoreGive()` is being called
- Check semaphore creation succeeded

### Problem: Audio data all zeros
- Verify GP7 (data) is receiving clock pulses on GP6
- Check ATSAMD21 microphone is sending data
- Verify GPIO functions are set correctly (`GPIO_FUNC_PIO0`)

### Problem: Audio sounds wrong (noisy)
- Increase decimation ratio (slower PDM clock)
- Add low-pass filter after PDM→PCM conversion
- Check for electrical noise on GPIO wires

### Problem: Buffer overflow/underflow
- Increase consumer task priority
- Reduce processing time in consumer task
- Create multiple consumer tasks if processing is too slow
